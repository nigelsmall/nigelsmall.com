Paths¶

Beyond simple nodes and relationships, more advanced structures come into play when writing Neo4j-based applications. The simplest of these is the path which is a linear sequence of nodes and relationships chained together end to end. Py2neo provides the Path class which is used to represent these structures:

class py2neo.neo4j.Path(*entities)¶

A chain of relationships.

>>> from py2neo import Node, Path, Rev
>>> alice, bob, carol = Node(name="Alice"), Node(name="Bob"), Node(name="Carol")
>>> abc = Path(alice, "KNOWS", bob, Rev("KNOWS"), carol)
>>> abc
({name:"Alice"})-[:KNOWS]->({name:"Bob"})<-[:KNOWS]-({name:"Carol"})
>>> abc.nodes
(({name:"Alice"}), ({name:"Bob"}), ({name:"Carol"}))
>>> abc.rels
(-[:KNOWS]->, <-[:KNOWS]-)
>>> abc.relationships
(({name:"Alice"})-[:KNOWS]->({name:"Bob"}), ({name:"Carol"})-[:KNOWS]->({name:"Bob"}))
>>> dave, eve = Node(name="Dave"), Node(name="Eve")
>>> de = Path(dave, "KNOWS", eve)
>>> de
({name:"Dave"})-[:KNOWS]->({name:"Eve"})
>>> abcde = Path(abc, "KNOWS", de)
>>> abcde
({name:"Alice"})-[:KNOWS]->({name:"Bob"})<-[:KNOWS]-({name:"Carol"})-[:KNOWS]->({name:"Dave"})-[:KNOWS]->({name:"Eve"})
create(*args, **kwargs)¶

Construct a path within the specified graph from the nodes and relationships within this Path instance. This makes use of Cypher’s CREATE clause.

get_or_create(*args, **kwargs)¶

Construct a unique path within the specified graph from the nodes and relationships within this Path instance. This makes use of Cypher’s CREATE UNIQUE clause.

nodes¶

Return a tuple of all the nodes which make up this path.

order¶

The number of nodes within this path.

relationships¶

Return a list of all the relationships which make up this path.

rels¶

Return a tuple of all the rels which make up this path.

size¶

The number of relationships within this path.

Paths can be returned from some Cypher queries and are used internally within create_path and get_or_create_path.
